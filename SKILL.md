# 赛马计划

## 先识别阶段

只加载当前阶段需要的参考文件，不要一次读取全部技能资料。

| 阶段 | 触发条件 | 必读文件 | 交付后动作 |
| --- | --- | --- | --- |
| 1 素材分析与策略 | 新产品、未确认方向、要求分析素材 | `references/stage1-codex.md`、`references/handoff-schema.md`、`references/validated-corrections.md` | 交付策略 Markdown、`phase1_handoff.json` 和阶段二接手指令后停止，等待确认 |
| 2 文案与提示词 | 用户已确认阶段一，或提供 `phase1_handoff.json` | `references/stage2-cheap-agent.md`、`references/handoff-schema.md`；包含详情页时再读 `references/detail-page-framework.md` | 交付 `phase2_output.json`、展示版 Excel 和三列生图交接表后停止，等待确认 |

阶段不明确时，优先根据现有文件判断：存在已确认的 `phase1_handoff.json` 进入阶段二；否则进入阶段一。若用户只要求第三阶段/批量生图，不加载阶段一、二参考文件，直接提示使用脚本 `scripts/generate_images_from_sheet.ps1` 或现有赛马专用入口。

## 阶段边界

1. 不跨越确认节点。
2. 阶段一必须完成视觉判断、事实核验、参考图选择和页面任务骨架；不能把重新读图的工作留给阶段二。
3. 阶段二默认禁止扫描原素材目录和重新读取图片；只消费交接包。交接包缺字段时返回缺口，不自行重做阶段一。
4. 第三阶段不使用技能推理生成或改写内容；只由脚本按用户确认的 Excel 工作表、列名和行号执行。
5. 医疗器械证照/备案原图如需展示，应作为上传参考图进入生图流程，并在提示词中要求把证照放入画框/展示框内、证照文字编号印章和版式不改；不要再用本地拼接的固定合成方式。非医疗器械产品不强制规划注册证或备案证明页。

## Token 节省规则

- 对图片建立稳定资产 ID；后续只用 ID 和简短摘要引用，避免重复描述长路径与画面。
- 阶段一读取图片时不裁剪、修图、重命名、移动或覆盖原素材；只生成临时总览和资产索引用于阅读。
- PDF 等文档先尝试用 MinerU 或同类转换工具生成 Markdown，再读取 Markdown 摘要和关键页；转换失败才读取原 PDF 关键页。
- 阶段一同时保存绝对路径和相对产品目录路径，便于本机和其他 Agent 使用。
- 将跨页面不变的产品锁定、视觉母版和合规规则放在交接包顶层；任务行只写差异。
- 阶段二只生成 `visible_copy` 和 `prompt_delta`，不重写产品事实、参考图摘要、完整提示词或链接策略。
- 完整生图提示词由 `compile_phase2_output.mjs` 确定性合并；低成本 Agent 不输出路径、全局产品锁定、视觉母版和通用禁用词。
- Excel 排版、路径检查、禁用策划词扫描和公式检查尽量交给脚本，不让 Agent 反复推理。
- 素材未变化时复用交接包；通过文件哈希或修改时间只补分析新增/变更素材。

## 共通硬规则

- 产品事实与电商卖点分开；每个卖点必须能回指产品事实和证据资产。
- 配方、成分、菌株、专利、检测报告等事实必须已被清晰证据核实才可进入可见文案；粗抽/OCR、文件名、旧链接文案或待复核字段只能进入复核备注或生成约束，不能当作主图卖点。
- 配方、资质、检测报告、执行标准和标签信息不能被包装成“编号清晰、报告明确、执行标准明确、标签清晰、检测结果摆出来”等内部核对型卖点。消费者可见文案必须转化为购买欲望和可感知收益，例如柔软、好用、好收纳、常备安心、场景更顺手；检测报告一般只做样张或关键指标局部的视觉佐证，不把报告编号、执行标准或检测结论直接做成标题/卖点卡/底部条。
- 可见文案必须直接面向消费者，不得出现“讲清、直接说明、围绕展开、证明价值”等内部策划措辞。
- 主图 `visible_copy` 和最终提示词里的“必须逐字显示”是重点审核对象：凡是设计说明、构图描述、卖点提取过程、参考图用途、注意事项、复核提醒、合规限制、模型负向约束，都不得进入主图可见卖点；这些内容只能进入 `prompt_delta`、`page_specific_constraints` 或 `review_notes`。
- “不作/不作为治疗承诺”“需人工复核”“使用前看说明书”等合规或复核提醒，不得作为成图可见卖点；需要控制风险时只写在提示词的当前页限制或复核备注中。
- 主图不得出现“植物乳杆菌配方线索”“配方线索”“配方清新双在线”等含糊或未核实表达；配方未核实时，主图只允许使用产品名、规格、口味、便携、0酒精等可核实卖点。
- 不同宣传方向的链接必须有可区分的场景、色调或设计风格。`visual_system` 只保留全局产品连续性；每条 `links[].visual_direction` 必须写出本链接自己的场景氛围、主色/辅助色、光感和信息组件方向，阶段二提示词必须按该链接方向展开，不得所有链接都套同一蓝色背景。
- 生图提示词短、准、只服务当前图片。发给 API 的提示词只保留三块：参考图说明、画面描述、页面中需要带出的文字。不要写任务编号、前后上下文、页面角色说明、执行模式、审核提醒或通用负向要求；“不要写医学护眼”“不要写承诺”等控制项放复核层，不进提示词主体。
- 五张主图都要有主文案与辅助解释；SKU 数量、组合、价格默认使用独立 SKU 图。多颜色产品的 SKU 图必须按颜色拆分，每个颜色一张 SKU 图，不得把所有颜色塞进同一张 SKU 图；全色系集合只用于详情页最后一屏或人审信息。
- `主图提示词` A1 必须为 `主图编号`，`详情页提示词` A1 必须为 `详情页编号`，顶部不加标题或空行。
- 脚本交接表必须只有三列表头：`说明`、`参考图的位置`、`生成提示词`；工作表名固定为 `生图任务表`，A1 必须为 `说明`。`说明` 只写链接序号、图片类型和组内序号，例如 `链接1｜主图｜第1/5张`。
- 尺寸和画面完整性必须前置在脚本中解决，不依赖生成后的视觉质检。第三阶段脚本必须在预览和正式执行前解析每行 `outputSize`；缺少明确尺寸时停止，不得默认方图、不得事后拉伸或裁切补救。
- 主图先按阶段二提示词生成 1440×1440 方图；一组 5 张主图方图完成后，第三阶段脚本默认用方图作为参考图生成对应 1440×1920 长图，要求文字、图标、产品和包装不改，只做比例适配的轻微排布调整。
- 第三阶段默认不让 Agent 做逐图质检，不生成用于人工审核的总览图；脚本只负责直接生成、记录基础状态，并把所有最终 PNG 汇总到同一个 `final_delivery` 文件夹，由用户人工质检。
- 第三阶段最终图片文件名必须以产品款式编码开头，例如 `JR0384_链接1_主图方图_第01张.png` 或脚本内部输出的 `JR0384_1_main_row002_...png`。款式编码优先取 `phase1_handoff.json` 的 `product.product_id`；直接跑表格脚本时自动从 Excel 文件名、输出路径或参考图路径识别，识别不到必须显式传 `-StyleCode`。
- 竞品素材必须标记为 `视觉参考` 或 `卖点参考`；不得作为我品事实证据，也不得上传到正式生图 API。

## 工具

- `scripts/validate_handoff.mjs`：校验阶段一交接包结构、资产引用和本地路径。
- `scripts/validate_phase2_output.mjs`：校验阶段二是否只输出增量、文案是否越界以及任务是否完整。
- `scripts/compile_phase2_output.mjs`：把交接包与阶段二增量合并为可用于 Excel 的完整任务，自动过滤竞品路径。
- `scripts/build_phase2_workbook.mjs`：使用 `@oai/artifact-tool` 生成并渲染校验标准 Excel。
- `scripts/build_generation_handoff_workbook.mjs`：从编译结果生成独立三列 `生图任务表`，供第三阶段脚本按固定表头和行号直接读取。
- `scripts/run_phase3_auto.ps1`：第三阶段全自动入口；只传 Excel 文件名或路径，脚本自动识别提示词工作表、编号列、参考图列、提示词列和数据行范围，并逐个工作表执行生图。带 `-PreviewOnly` 只预览不调用 API。
- `scripts/run_phase3_wizard.ps1`：第三阶段交互入口；适合在 Cursor/PowerShell 终端直接运行，按提示选择 Excel、工作表、参考图列、提示词列和行号范围，默认先预览，确认后才调用 API。
- `scripts/generate_images_from_sheet.ps1`：通用第三阶段底层入口；按用户指定的工作表、参考图列、生图提示词列和行号读取 Excel，先预览，带 `-Execute` 才调用 API。
- `scripts/extract_sheet_image_tasks.mjs`：通用 Excel 抽取器，供第三阶段脚本使用。
- `scripts/compose_fixed_assets.ps1`：旧版固定资产本地合成脚本，仅用于历史文件回看；新流程默认不用它生成证照页。
- `scripts/run_stage3_automated.ps1`：赛马专用第三阶段入口；保留给已验证的交接包/阶段二输出流程。
- `scripts/extract_main_image_prompts.mjs`：抽取用户确认的 Excel 行。
- `scripts/saima_generate_images.ps1`：底层生图执行器；可从赛马 Excel 或通用抽取任务 JSON 调用 API，并输出原图、最终图、元数据、`final_delivery` 汇总文件夹和运行摘要；总览图仅在显式传 `-CreateContactSheets` 时生成；默认在每组 5 张主图方图完成后追加生成 1440×1920 长图，传 `-SkipMainLongImages` 可跳过。

技能规则发生冲突时，按以下优先级处理：用户本轮明确修改 > 用户确认的阶段交付物 > `validated-corrections.md` > 当前阶段参考文件 > 通用规则。
