# 阶段一：Codex 素材分析与交接包

若用户指定“沃朗森风格”或同义表达，先额外读取 `wolansen-style.md`，并把该预设写入策略 Markdown 与交接包 `visual_system`。所有页面绑定匹配产品形态的第一身份参考，链接差异不得破坏沃朗森蓝白母版和低复杂度产品优先原则。

若用户指定“振德风格”或同义表达，先额外读取 `zhende-style.md`，并把该预设写入策略 Markdown 与交接包 `visual_system`。所有页面绑定匹配包装形态、规格、颜色、尺码和型号的第一身份参考，链接差异不得破坏振德标准蓝白、粗黑标题、单一证据和底部规格栏母版。

## 目标

一次完成高成本视觉理解，把事实、链接方向、参考图选择和页面任务压缩为可复用交接包。阶段二不得再读全部原图。

## 执行顺序

1. 检查产品目录及平台通用素材合规要求。
2. 盘点我品、现有链接、竞品、SKU、包装、说明书和资质；图片原文件不裁剪、不修图、不重命名、不移动，只生成临时总览图和索引用于阅读。
3. 对 PDF 等文档资料先尝试转换为 Markdown：优先使用 MinerU；转换失败时再读取 PDF 关键页。转换产物放在输出目录的 `converted_documents/`，并在资产卡或策略 Markdown 中记录来源 PDF、Markdown 路径和转换状态。
4. 先生成总览图，去除重复或近似图片的分析优先级，再打开关键原图；“去除”只表示分析时不重复读取，不删除或改动素材。
5. 读取每张关键图时同步建立资产卡，不要在分析结束后重新回看图片补摘要。
6. 竞品先标记 `视觉参考` 或 `卖点参考`。视觉参考只学构图、色调、比例、角度、光影和质感；卖点参考只学表达结构，所有主张回到我品证据。
7. 建立产品事实、证据来源、风险和消费者价值。看不清的数字、证号、日期写“需人工复核”，不得猜测。
8. 按“目标人群 + 真实需求 + 使用场景”拆链接；一条链接只设一个宣传方向，并为每条链接指定可区分的场景色调和视觉风格。
9. 为每条链接建立主图、SKU 图和详情页页面地图；逐页选择参考资产并写清用途。
10. 同时输出策略 Markdown、`phase1_handoff.json` 和 `stage2_agent_instruction.txt`，运行校验脚本。接手指令只允许阶段二 Agent 读取交接 JSON 和阶段二规则。
11. 交付后停止，等待用户确认；不要生成阶段二提示词或调用 API。

## 策略 Markdown 必须包含

- 产品基础信息、资质、SKU 和证据来源；
- 竞品参考分类表；
- 文档转换记录：PDF/说明书/资质原文件、Markdown 转换路径、转换工具、失败原因或需人工复核项；
- 人群、需求、场景；
- 产品事实 → 用户麻烦 → 消费者结果 → 视觉证据；
- 链接宣传策略；
- 链接视觉差异表：每条链接的场景氛围、主色/辅助色、光感、背景元素和信息组件方向；不同链接不得全部使用同一套蓝白水感背景；
- 主图、SKU 图和详情页页面地图；
- 选中参考图索引：资产 ID、路径、画面摘要、参考类型、用途、不可复制风险；
- 阶段二交接说明及 JSON 路径；
- 合规边界和待人工复核项。

## 交接深度

阶段一必须让未看过图片的 Agent 能回答：

1. 当前任务应该使用哪几张参考图；
2. 第一张上传图为什么是产品身份基准；
3. 每张图只说什么，必须显示什么；
4. 哪些文字、规格、Logo、证号和版式不可改；
5. 哪些竞品图只能学习视觉或卖点结构，不能上传；
6. 哪些页面需要人工或 Codex 复核。

如果交接包无法回答任一问题，阶段一未完成。

阶段一还必须：

- 在 `stage2_contract.forbidden_visible_terms` 中列出本产品容易被低成本 Agent 过度扩写的词；
- 对规格、证号、SKU组合等必须逐字出现的信息设置任务级 `visible_copy_required`；
- 把消费者文案审核要求与图像模型负向限制分开，不让内部审核语言进入可见文案。
- 在每条 `links[].visual_direction` 中写明本链接区别于其他链接的视觉方向。全局视觉母版只用于保持产品连续性，不能替代链接级差异。
- 主图任务的 `copy_brief` 只能写消费者可见目标，不得把设计说明、卖点提取过程、注意事项、复核提醒或合规限制写成主图可见目标。

## 五张主图默认分工

普通个护、日百和卫生用品：

1. 场景需求点击图；
2. 材质证明图；
3. 成分/结构证明图；
4. 使用体验图；
5. 核心解决方案图。

SKU 数量、组合、价格和优惠使用独立 SKU 图。多颜色产品的 SKU 图按颜色拆分，每个颜色一张 SKU 图；不要把所有颜色合在同一张 SKU 图里。全色系集合可放在详情页最后一屏。医疗器械有清晰证照且用户确认需要展示时，可规划证照信任页；证照原图作为上传参考进入生图流程，画面用画框/展示框承载证照，并在提示词中严格限定证照文字、编号、印章和版式不改。非医疗器械产品不固定规划注册证或备案证明页。

## 素材缓存

交接包中的资产卡即最小视觉缓存。后续运行先核对路径、文件大小和修改时间；未变化的资产直接复用摘要，只分析新增或变化图片。需要更强缓存时可补 SHA-256，但不要为了计算哈希重新进行视觉分析。
