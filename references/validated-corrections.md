# 赛马计划已验证纠错清单

本清单来自真实赛马计划执行与返工。每次启动赛马计划时先读取，并把下列规则视为交付门槛，不要把它当作可选建议。

## 1. 链接规划

- 按“目标人群 + 真实需求 + 使用场景”拆链接，不要默认按包装大小或规格拆链接。
- 只规划用户确认的产品形态和 SKU。主推迷你便携装时，不要混入 80 片大规格或其他未确认规格。
- 同一产品可从学生上学、爱干净女士、通勤出差、外出旅行、日常物表清洁等动机拓展，但每条链接只能有一个清晰主方向。
- 酒精湿巾不只表达消毒，也可表达一般物表和日常小污渍清洁；使用场景必须符合标签与素材证据。
- 第一阶段的重点是“链接方向对应哪条卖点表达”。卖点与方向不对应时，移动到其他链接或删除。
- 第一阶段必须同时区分链接视觉方向。不同宣传方向不能只换文案、不换画面气质；每条链接要有自己的场景、主色/辅助色、光感和背景元素。

## 2. 消费者可见文案

- 标题、副标题、卖点卡和底部条必须是卖家直接对消费者说的话，不是给设计师的页面说明。
- 禁止使用“主要成分先讲清、直接说明、不含糊、材质值得看、围绕场景展开、证明随身价值、使用理由才成立、单片大小也要讲清”等内部动作语言。
- 成分页使用“关键成分/含量事实”做主标题；副标题和卖点卡说明成分作用、当前场景价值或消费者收益，不评价是否讲清楚。
- 材质页使用“材质事实 + 可感知使用结果”；尺寸页使用“真实尺寸 + 对擦拭或携带的帮助”。
- 基础信息页只保留消费者真正关心的片数、单片尺寸、主要成分、保质期、使用或适用判断。执行标准编号、常规储存要求等默认不占核心画面。
- 配方、成分、菌株、专利、检测报告等内容必须有清晰证据才能进入可见文案。粗抽/OCR、文件名、旧链接文案或待复核字段只能放在复核备注或生成约束里，不得包装成主图卖点。
- 配方、资质、检测报告、执行标准和标签信息不是天然卖点，不能写成“报告编号摆出来、编号清晰、报告明确、执行标准明确、标签清晰、检测结果清楚”等内部核对语言。报告类素材一般只做样张或关键指标局部放大，消费者文案必须转化成柔软、好用、好收纳、常备安心、场景更顺手等购买理由。
- “不作治疗承诺”“不作为治疗承诺”“需人工复核”“使用前看说明书”是合规或复核提醒，不是消费者卖点；如需控制风险，只能写进提示词限制或复核备注，不能作为画面可见文字。
- 主图 `visible_copy` 和最终提示词里的“必须逐字显示”必须单独审核。设计说明、构图描述、卖点提取过程、参考图用途、注意事项、复核提醒、合规限制和模型负向约束，都不能出现在主图可见卖点上。
- 每条可见文案都做上架测试：如果消费者看完仍不知道产品事实、场景价值或购买理由，就重写。

## 3. 竞品参考分类

- 读取竞品前先标注为 `视觉参考` 或 `卖点参考`；同一素材兼具两种用途时拆成两条分析记录。
- `视觉参考` 读取主体画面、整体色调、颜色搭配、设计质感，以及产品展示的构图、比例、角度、光影和场景组织。
- `卖点参考` 读取文字内容、卖点提炼与表达、卖点证据链路、卖点呈现方式，以及画面体现的人群、需求和场景。
- 视觉参考不能作为我品卖点证据；卖点参考不能直接复制竞品结论、数字或功效，必须回到我品素材核验。
- 第一阶段输出竞品参考分类表；第二阶段在 `竞品结构参考` 与 `参考图用途` 中明确写出参考类型。

## 4. 五张主图

- 普通卫生用品默认顺序：场景需求、材质、成分/结构、使用体验、便携或核心解决方案。
- 五张主图都要有主文案和辅助解释，不要只有第一张有内容。
- 购买选择、SKU数量、组合、价格和优惠不放进五张主图，统一进入 SKU 图。多颜色产品要按颜色拆 SKU：每个颜色一张 SKU 图，不要把所有颜色合成一张 SKU 图；详情页最后一屏可以做全色系集合展示。
- 第 5 张不要机械加入“购买选择”。医疗器械存在清晰可核验的注册证/备案原图且用户确认需要展示时，可规划证照内容；非医疗器械产品不要固定规划注册证或备案证明页。
- 医疗器械证照页不再使用本地拼接合成。证照原图作为上传参考图进入生图流程，提示词必须要求证照置于画框/展示框内，主要文字、编号、印章、表格和字段关系保持参考图原样，只允许扩展外围背景与辅助标题。
- 主图文案密度默认参考竞品：大主标题、强点击理由、3-4 个辅助卖点卡、底部信任条；缩略图下仍要先看见产品和主标题。
- 主图不得出现“植物乳杆菌配方线索”“配方线索”“配方清新双在线”等含糊或未核实表述。配方未被人工确认时，主图优先使用产品名、规格、口味、便携、0酒精、使用场景等可核实购买理由。
- 主图可以通过“当前页限制”要求模型避免治疗承诺、避免具体菌株或专利证书正文，但不得把这些限制生成到画面文字上。
- 同一链接的 5 张主图保持统一视觉系统；不同链接之间必须能从背景、色调、场景元素或设计组件上看出差异，不得全部使用同一蓝色水感模板。

## 5. 详情页

- 先从竞品详情页识别页面角色，再建立本品页面地图；不要只总结图片内容。
- 每屏都写清本屏角色、承接上一屏、消费者可见文案、视觉证据和下一屏逻辑。
- 同一链接保持统一色系、标题层级、信息卡样式、产品锚点和场景人群；不同链接不要混用人物与场景。
- 详情页不能把“产品资料目录”直接变成页面顺序。先讲场景需求，再用材质、成分、体验、尺寸和便携逐步回答购买顾虑。

## 6. Excel 与阶段确认

- 第一阶段只交付策略 Markdown；确认后才生成第二阶段 Excel；再次确认工作表和起止行后才调用 API。
- `主图提示词` 的 A1 必须是 `主图编号`，`详情页提示词` 的 A1 必须是 `详情页编号`；顶部不得有标题行、空白行或说明行。
- SKU 图使用独立工作表，不占五张主图行号。
- 第三阶段必须复述工作表、Excel 起止行、有效数量和目标尺寸。“前 5 张”对应表头后的 Excel 第 2-6 行。
- 始终读取用户最后确认的修订版文件；不要因为旧文件名更短而误用旧版。

## 7. 产品身份参考

- 第一张上传图是产品身份基准，必须与当前画面要生成的产品形态一致；其他参考图只提供构图、动作和场景。
- 生成外盒时使用清晰外盒图作第一参考；生成单包时必须使用清晰单包图作第一参考。不要用同时包含外盒和单包的组合图作为单包唯一身份基准，否则模型容易把“8片×8包”等外盒规格印到单包上。
- 第一身份参考使用原始 PNG 或高质量原图、高细节上传；不要压成低清 JPEG。场景参考可使用低细节以控制请求体大小。
- 把包装结构、恒品 logo、产品名、75% 标识、片数规格、主要图标、色块和相对位置视为不可改写的印刷内容。
- 提示词只能降低错误概率，不能替代质检。不要使用固定产品贴图强行合成到场景：它会受角度限制，且透视、遮挡和光影难以自然匹配。
- 没有与目标形态和角度匹配的清晰产品图时，停止该行并要求补素材，不要反复消耗 API 猜包装。

## 8. API 生成与返工

- 品牌商品参考图请求失败时，不得静默回退到无参考图生成。保持相同参考图自动重试；重试耗尽后停止该行。
- 发生单图问题时按原 Excel 行号重跑，使用 `-AdditionalPrompt` 限定产品数量、包装形态、可见面、透视和不得新增的结构；不要改写已确认文案。
- 包装数量越多、透视越复杂、遮挡越严重，身份漂移风险越高。返工时优先减少同屏产品数量、放大核对面、降低极端视角。
- 迷你扁平密封包装不得被改成抽取式大包、翻盖湿巾或盒装抽纸；不得在包装上新增“随身携带、一抽一张”等画面卖点文字。

## 9. 尺寸前置防错与汇总交付

- 以 Excel `成图尺寸` 为准。主图常用 1440×1440，详情页测试常用 1440×1920，但不得用默认方图覆盖表格尺寸。
- 正式请求前和日志中都要核对尺寸解析结果。详情页被解析成方图时立即停止，不能事后拉伸、硬裁切或补边冒充竖图。
- 尺寸、比例和画面完整性必须靠脚本前置防错，不依赖 Agent 生成后逐图质检；缺少 `outputSize` 时直接报错并修交接数据。
- 默认不让 Agent 逐张检查品牌名、产品名、规格、主要图标、包装色块、外部品牌、二维码、联系方式和平台名；脚本只汇总最终图，交由用户人工质检。
- 产品包装出现乱码、规格串用、结构变形、虚构翻盖或新增印刷文字时，直接判废。
- 脚本把所有最终 PNG 汇总到同一个 `final_delivery/` 文件夹。总览图、rejected 目录和人工质检截图不默认生成，除非用户明确要求。
- “保证产品正确”来自高质量参考、脚本前置尺寸防错和用户人工质检，不来自一句强提示词，也不要求 Agent 默认逐图质检。
- 医疗器械证照页若出现证号、日期、主体名称、印章、表格字段的改写、乱码、缺失或新增，直接判废；不能用“看起来像证书”的 AI 图代替上传证照参考图中的真实信息。
